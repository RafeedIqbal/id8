# ID8 Product Guidelines

## Prose and Documentation Style
The AI-generated documentation (PRDs, Tech Plans, etc.) should follow a **Concise & Functional** tone.
- **Brevity:** Use bullet points and short, impactful sentences.
- **Structure:** Organize information with clear headings and subheadings.
- **Clarity:** Avoid technical jargon where simpler terms suffice, but maintain precision when describing architectural details.
- **Directness:** Focus on "what" and "why" without excessive preamble.

## Visual and Aesthetic Standards
Generated applications should embody a **Modern & Minimalist** aesthetic.
- **Whitespace:** Use generous spacing to reduce cognitive load and create a clean look.
- **Hierarchy:** Establish a clear visual hierarchy using typography and subtle color variations.
- **Minimalism:** Prioritize essential elements and avoid unnecessary decorative features.
- **Modernity:** Use contemporary design patterns, such as soft shadows, subtle gradients, and rounded corners.

## User Experience (UX) Principles
The core UX principle is **Feedback-Driven Interaction**.
- **Interactive Feedback:** Provide immediate visual cues for all user actions (e.g., loading states, success/error messages, hover effects).
- **Predictability:** Ensure consistent behavior across all interactive components.
- **Usability:** Design interfaces that are intuitive and require minimal learning for the user.

## Design and Frontend Standards
All generated applications must adhere to the following technical design standards:
- **Tailwind-First Utility UI:** Use Tailwind CSS 4 as the primary styling framework, prioritizing utility classes for rapid development and consistency.
- **Consistent Componentry:** Build and reuse a standardized library of UI components (buttons, inputs, cards, etc.) to ensure a unified look and feel across the entire application.
- **Responsive Design:** Ensure all layouts are fully responsive and function correctly across various screen sizes.
